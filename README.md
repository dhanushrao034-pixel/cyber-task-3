# Task 3 – Basic Vulnerability Scan on Local Machine

## 🎯 Objective
To perform a basic vulnerability assessment on my local machine using a free vulnerability scanning tool and analyze identified security risks.

## 🛠 Tool Used
- OpenVAS Community Edition
- Nessus Essentials

## 🔍 Scan Configuration
- **Target:** Localhost / Local Machine IP
- **Scan Type:** Full Vulnerability Scan
- **Duration:** Approximately 45 minutes

## 📊 Scan Results Summary

| Severity | Count |
|--------|--------|
| Critical | 2 |
| High | 4 |
| Medium | 6 |
| Low | 3 |

## 🚨 Top Critical Vulnerabilities Identified
- Outdated software packages
- Missing security patches
- Weak SSL/TLS configuration
- Open ports with exposed services

## 📈 CVSS (Common Vulnerability Scoring System)

CVSS is used to measure the severity of vulnerabilities on a scale from **0 to 10**.

| Score Range | Severity |
|-------------|-----------|
| 0 – 3.9 | Low |
| 4.0 – 6.9 | Medium |
| 7.0 – 8.9 | High |
| 9.0 – 10 | Critical |

## 🛡 Remediation Steps
- Updated outdated software
- Applied missing system patches
- Disabled unnecessary services
- Configured firewall rules properly

Screenshots of the vulnerability scanning process and results are available in the **screenshots/** folder.

This task helped in understanding how vulnerability scanners identify security issues and how organizations can mitigate risks through proper patching, configuration management, and continuous monitoring.
